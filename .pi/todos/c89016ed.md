## Description
Some strings embed dynamic values via interpolation. Languages with different word order need positional arguments, not concatenation.

## Strings Needing Attention
1. **ModelManagerView.swift:45** — `Text("Remove \(model.name) from device.")` → word order varies by language
2. **GitView.swift:338** — `Text("@\(auth.username.isEmpty ? "authenticated" : auth.username)")` → format varies
3. **GitView.swift:341** — `Text("via \(auth.method == .oauth ? "GitHub OAuth" : "Personal Access Token")")` → connector word varies
4. **ChatView.swift:259** — `Text(workingDir.workingDirectoryURL != nil ? "CHANGE" : "OPEN")` → conditional strings (already fine, just ensure both keys are in catalog)
5. **ChatView.swift:180** — `Text(llmService.isModelLoaded ? "READY" : "NO MODEL")` → same pattern

## Pattern
```swift
// Before
Text("Remove \(model.name) from device.")

// After — use String(localized:) with interpolation (auto-handled by String Catalog)
Text("Remove \(model.name) from device.")
// The String Catalog will extract this with %@ placeholder, allowing translators to reorder
```

## Notes
- SwiftUI `Text` with interpolation auto-generates positional placeholders in the String Catalog
- Just make sure these strings are present in the catalog with translator comments explaining what the variable is
