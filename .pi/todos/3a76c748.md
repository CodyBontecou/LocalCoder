Diagnosed model-load crash as likely MLX/iOS memory pressure / jetsam when loading large installed models. Implemented safeguards in `LocalCoder/Services/LLMService.swift`:

- switched load path to use resolved local MLX folder URLs when available
- added MLX memory tuning (`Memory.cacheLimit`) and cache clearing on unload/failure
- added preflight size check based on `.safetensors` weight size vs device working-set budget
- added detailed runtime diagnostics and memory snapshots to debug log
- surfaced friendly `modelTooLargeForDevice` error instead of silent failure

Updated `LocalCoder/Views/ModelManagerView.swift` to:
- show load failures in an alert
- show the last load error in the status subtitle
- explain that installed models can still exceed device memory budgets

Validation: `xcodebuild -scheme LocalCoder -project LocalCoder.xcodeproj -destination 'generic/platform=iOS' build CODE_SIGNING_ALLOWED=NO` succeeded.
