## Description
Currently only 1 accessibility label exists in the entire app (`.accessibilityLabel("Hide keyboard")` in ChatView). All interactive elements — buttons, icons, swipe actions — need accessibility labels for both a11y and localization.

## Areas Needing Labels
- **ChatView.swift**: History button, new chat button, debug button, model button, send/stop button, folder picker, close error button
- **ConversationListView.swift**: Close button, new conversation button, delete swipe action
- **GitView.swift**: All icon-only buttons (sign in, copy code, clone, etc.)
- **FilesView.swift**: Context menu actions, navigation items
- **ModelManagerView.swift**: Delete, download, unload buttons
- **CodeEditorView.swift**: Close, font size picker, save button
- **DebugPanelView.swift**: Auto-scroll toggle, clear button

## Notes
- Accessibility labels are localizable when using `Text()` or `LocalizedStringKey`
- This improves both VoiceOver support and localization coverage
